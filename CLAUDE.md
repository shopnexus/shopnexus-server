# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & Run

```bash
make dev          # hot-reload dev server (air)
make run          # direct server run
make build        # compile to bin/server
make generate     # run all go:generate (restate proxies, etc.)
make pgtempl      # generate SQLC query templates for all modules
make migrate      # run database migrations
make seed         # seed data
make register     # register services with Restate admin API
```

All `cmd/` packages use directory paths (`go run ./cmd/pgtempl/`) not file paths, since some have multiple files.

**Environment**: `GOLANG_PROTOBUF_REGISTRATION_CONFLICT=warn` is required (set in Makefile) due to proto file collision between restate-sdk and milvus.

**Infrastructure**: `docker compose -f deployment/docker-compose.yml up -d` starts PostgreSQL, Redis, NATS, Restate (3-node cluster), Milvus, MinIO.

## Architecture

Social marketplace modular monolith designed for microservice extraction via Restate. Entry point: `cmd/server/main.go` → `fx.New(app.Module).Run()`.

**No customer/vendor distinction** — any account can both buy and sell. Orders track buyer (`customer_id`) and seller (`vendor_id`) per transaction.

### Module Structure (Vertical Slice)

Each module under `internal/module/<name>/` follows this layout:

```
biz/
  interface.go      # XxxBiz interface + XxxHandler struct + constructor + go:generate directive
  restate_gen.go    # Auto-generated Restate HTTP client (DO NOT EDIT)
  *.go              # Business logic methods (use restate.Context)
db/
  migrations/       # SQL schema (*.up.sql / *.down.sql)
  queries/          # SQLC query templates (pgtempl-generated + *_custom.sql for hand-written)
  sqlc/             # Generated DB code (DO NOT EDIT)
model/
  *.go              # DTOs, domain models, error sentinels
transport/echo/
  *.go              # HTTP handlers
fx.go               # Uber fx module wiring
```

### Naming Convention

| Thing | Name | Example |
|-------|------|---------|
| Interface (public API) | `XxxBiz` | `AccountBiz` |
| Struct (implementation) | `XxxHandler` | `AccountHandler` |
| Restate HTTP client (generated) | `XxxRestateClient` | `AccountRestateClient` |
| Constructor | `NewXxxHandler` | `NewAccountHandler` |
| Service name (Restate) | via `ServiceName()` method | `"Account"` |

### Restate Integration Pattern

1. **`interface.go`** defines `XxxBiz` interface + `XxxHandler` struct + `ServiceName()` method + `//go:generate` directive.
2. **`restate_gen.go`** is auto-generated by `cmd/genrestate/` — implements `XxxBiz` via HTTP calls to Restate ingress.
3. **`fx.go`** provides both `*XxxHandler` (for Restate registration) and `XxxBiz` (proxy, for cross-module deps and transport).
4. **Transport handlers** depend on `XxxBiz` (interface), not `*XxxHandler` (concrete).
5. **Cross-module biz deps** also use `XxxBiz` interface (resolved to Restate proxy by fx).
6. **`app/restate.go`** registers all Handler structs with `restate.Reflect()` and auto-registers with Restate admin on startup.

**Biz methods use `restate.Context`** (not `context.Context`). This is required for `restate.Reflect()`. Exception: cron/init methods that use `context.Background()` stay as `context.Context` and call other methods through the Restate HTTP ingress client (`restateClient` field) instead of directly.

**Fire-and-forget calls**: `restate.ServiceSend(ctx, "ServiceName", "MethodName").Send(params)` — durable, exactly-once delivery via Restate. Used for notifications, analytics tracking.

**Inside `restate.Run()` closures**, `ctx` is `restate.RunContext` which can't be passed to methods expecting `restate.Context` — move cross-module calls outside `Run()` blocks.

**`restate.Run()` serialization warning**: Return values from `Run()` are serialized to JSON in Restate's journal. Types like `uuid.UUID` (byte array), `null.*`, and `int64` may not round-trip correctly. Avoid returning complex DB structs from `Run()`. Keep DB reads outside `Run()` when possible.

**Init-time calls**: Never call through the Restate proxy in constructors — Restate isn't running yet. Use background goroutines or private `context.Context` helpers for setup calls (e.g., `SetupPaymentMap`, `SetupObjectStore`).

### Database Layer

- **pgsqlc**: `pgsqlc.Storage[T]` wraps pgx pool with transaction support.
- **SQLC**: generates type-safe Go from SQL queries. Config in `sqlc.yaml`. Uses `guregu/null/v6` for nullable types, `pgx/v5` driver.
- **pgtempl** (`cmd/pgtempl/`): generates SQLC query templates from migration files. Always use `-skip-schema-prefix` flag to get clean query names (e.g., `GetAccount` not `GetAccountAccount`). Custom queries go in `*_custom.sql` files (not overwritten by pgtempl).
- Each module has its own PostgreSQL schema (e.g., `account.*`, `catalog.*`, `order.*`).
- pgtempl only handles `CREATE TABLE` and `ALTER TABLE ADD COLUMN` — it does NOT handle `DROP COLUMN`. After dropping columns, manually remove them from generated queries.

## Code Generation

```bash
# Regenerate SQLC queries (always use -skip-schema-prefix)
go run ./cmd/pgtempl/ -module <name> -skip-schema-prefix
# then: sqlc generate

# Regenerate Restate proxies after changing interface.go
go generate ./internal/module/<name>/biz/

# Regenerate all
make generate
```

The `genrestate` tool reads `interface.go`, finds the interface, and generates a Restate client struct. Methods must have exactly 1 input param (besides ctx). Multi-param methods must be wrapped in a params struct or made private.

## Conventions

- **Import alias**: use `sharedmodel` for `internal/shared/model` (not `commonmodel`).
- **Error pattern**: Client-visible errors use `sharedmodel.NewError(httpStatusCode, "message")` in `model/error.go`. Return with `.Terminal()` to prevent Restate retries. Internal errors use `fmt.Errorf("action: %w", err)` — no "failed to" prefix.
- **Terminal errors**: Always use `.Terminal()` when returning user-facing errors (validation, not found, conflict). Use `sharedmodel.WrapErr(msg, err)` instead of `fmt.Errorf` when wrapping errors that might be terminal — it preserves the terminal flag.
- **Nullable types**: `guregu/null/v6` for DB fields, `*T` pointers for JSON serialization in models exposed to Restate (generic types like `null.Value[T]` break Restate's JSON schema generation).
- **Pointer helpers**: `pgutil.Ptr[T](val)` and `pgutil.NullToPtr[T](val, valid)` in `internal/shared/pgutil/`.
- **Config**: Viper-based, loaded via `config.GetConfig()`. Struct in `config/struct.go`, defaults in `config/config.default.yml`.
- **Geocoding**: `internal/infras/geocoding/` with `Client` interface. Currently uses Nominatim (OpenStreetMap). Swap provider in `app.go` `NewGeocodingProvider()`.
- **LLM/Embedding**: `internal/infras/llm/` with `Client` interface. Supports Python, OpenAI, Bedrock providers. Configured via `config.default.yml` `llm` section.

## Commit Rules

- **Short messages, no description body, no Co-Authored-By**.
- Imperative mood: `add`, `fix`, `refactor`, `rename`, `update`, `remove` — not `added`, `fixes`, `updating`.
- Examples: `add notifications feature`, `fix checkout item creation`, `rename structs to XxxHandler`.
- When recommitting (squash/rebase), group logically: one commit per feature/refactor, not per file.
