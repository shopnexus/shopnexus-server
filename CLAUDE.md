# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & Run

```bash
make dev          # hot-reload dev server (air)
make run          # direct server run
make build        # compile to bin/server
make generate     # run all go:generate (restate proxies, etc.)
make pgtempl      # generate SQLC query templates for all modules
make migrate      # run database migrations
make seed         # seed data
make register     # register services with Restate admin API
```

All `cmd/` packages use directory paths (`go run ./cmd/pgtempl/`) not file paths, since some have multiple files.

**Environment**: `GOLANG_PROTOBUF_REGISTRATION_CONFLICT=warn` is required (set in Makefile) due to proto file collision between restate-sdk and milvus.

**Infrastructure**: `docker compose -f deployment/docker-compose.yml up -d` starts PostgreSQL, Redis, NATS, Restate (3-node cluster), Milvus, MinIO.

## Architecture

Modular monolith designed for microservice extraction via Restate. Entry point: `cmd/server/main.go` → `fx.New(app.Module).Run()`.

### Module Structure (Vertical Slice)

Each module under `internal/module/<name>/` follows this layout:

```
biz/
  interface.go      # Client interface + struct + constructor + go:generate directive
  restate_gen.go    # Auto-generated Restate HTTP proxy (DO NOT EDIT)
  *.go              # Business logic methods
db/
  migrations/       # SQL schema (*.up.sql / *.down.sql)
  queries/          # SQLC query templates
  sqlc/             # Generated DB code (DO NOT EDIT)
model/
  *.go              # DTOs, domain models, error sentinels
transport/echo/
  *.go              # HTTP handlers
fx.go               # Uber fx module wiring
```

### Restate Integration Pattern

The key pattern for microservice-readiness:

1. **`interface.go`** defines `XxxClient` interface + `XxxBiz` struct. Has `//go:generate` directive.
2. **`restate_gen.go`** is auto-generated by `cmd/genrestate/` — implements `XxxClient` via HTTP calls to Restate ingress.
3. **`fx.go`** provides both `*XxxBiz` (for Restate registration) and `XxxClient` (proxy, for transport layer).
4. **Transport handlers** depend on `XxxClient` (interface), not `*XxxBiz` (concrete) — enabling transparent local/remote switching.
5. **`app/restate.go`** registers all Biz structs with `restate.Reflect()` and auto-registers with Restate admin on startup.

**Biz methods use `restate.Context`** (not `context.Context`). This is required for `restate.Reflect()`. Exception: cron/init methods that use `context.Background()` stay as `context.Context` and call other Biz methods through the Restate HTTP ingress client instead of directly.

**Restate service calls** for fire-and-forget: `restate.ServiceSend(ctx, "ServiceName", "MethodName").Send(params)`.

### Cross-Module Dependencies

Modules hold direct `*XxxBiz` pointers for internal calls (not through proxy). The order module depends on account, catalog, inventory, promotion, analytic. Inside `restate.Run()` closures, `ctx` is `restate.RunContext` which can't be passed to methods expecting `restate.Context` — move cross-module calls outside `Run()` blocks.

### Database Layer

- **pgsqlc**: `pgsqlc.Storage[T]` wraps pgx pool with transaction support
- **SQLC**: generates type-safe Go from SQL queries. Config in `sqlc.yaml` (9 modules). Uses `guregu/null/v6` for nullable types, `pgx/v5` driver.
- **pgtempl** (`cmd/pgtempl/`): generates SQLC query templates from migration files. `go run ./cmd/pgtempl/ -module all` for all modules.
- Each module has its own PostgreSQL schema (e.g., `account.*`, `catalog.*`, `order.*`).

## Code Generation

```bash
# Regenerate Restate proxies after changing interface.go
go generate ./internal/module/<name>/biz/

# Regenerate SQLC queries after changing migrations
go run ./cmd/pgtempl/ -module <name>   # then: sqlc generate

# Regenerate all
make generate
```

The `genrestate` tool reads `interface.go`, finds the interface, and generates a proxy struct. Methods must have exactly 1 input param (besides ctx). Multi-param methods must be made private or wrapped in a params struct.

## Conventions

- **Import alias**: use `sharedmodel` for `internal/shared/model` (not `commonmodel`)
- **Error pattern**: client-visible errors use `sharedmodel.NewError("module.code", "message")` in `model/error.go`. Internal errors use `fmt.Errorf("action: %w", err)` — no "failed to" prefix.
- **Nullable types**: `guregu/null/v6` for DB fields, `*T` pointers for JSON serialization in models exposed to Restate (generic types like `null.Value[T]` break Restate's JSON schema generation).
- **Pointer helpers**: `pgutil.Ptr[T](val)` and `pgutil.NullToPtr[T](val, valid)` in `internal/shared/pgutil/`.
- **Config**: Viper-based, loaded via `config.GetConfig()`. Struct in `config/struct.go`, defaults in `config/config.default.yml`.
